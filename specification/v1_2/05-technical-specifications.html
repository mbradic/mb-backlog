5. Technical Specifications

Architecture

The mb-backlog application follows a three-tier architecture consisting of:

  • Presentation Layer: Web-based user interface built with modern frontend framework
  • Application Layer: RESTful API backend handling business logic and data operations
  • Data Layer: Relational database for persistent storage of user data and backlog items

Technology Stack

Layer Technology Version Justification
Frontend React / Vue.js / Angular Latest LTS Modern framework for interactive UI with strong community support
Backend Node.js / Python / Java Latest LTS Scalable runtime with proven production track record
Database PostgreSQL / MySQL Latest Stable Reliable relational database with strong ACID compliance
Hosting Cloud Platform (AWS/Azure/GCP) N/A Scalable infrastructure with managed services

Database Schema (Summary)

Main entities:

  • Users: User account information, authentication credentials, profile data
  • BacklogItems: Core backlog item data including title, description, priority, status, timestamps

API Specifications

The backend exposes RESTful API endpoints for CRUD operations on backlog items and user management. Key endpoints include:

  • POST /api/auth/register - User registration
  • POST /api/auth/login - User authentication
  • GET /api/backlog - Retrieve all backlog items with filtering and sorting parameters
  • POST /api/backlog - Create new backlog item
  • PUT /api/backlog/{id} - Update backlog item
  • DELETE /api/backlog/{id} - Delete backlog item
  • GET /api/backlog/search - Search backlog items with filtering and sorting capabilities

All API responses use JSON format with standard HTTP status codes.

Integration Points

  • Authentication Provider: OAuth 2.0 integration for third-party authentication (optional for v1.2)
  • Email Service: Email notifications for account activities and reminders (future enhancement)
