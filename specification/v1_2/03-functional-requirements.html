3. Functional Requirements

The following section details the specific functional requirements for mb-backlog v1.2. Each requirement is assigned a unique identifier for traceability.

FR-001: User Authentication and Account Management

Description: The system shall provide user authentication mechanisms to secure personal backlog data. Users must be able to create accounts, log in, and manage their profile information.

Related User Role: All Users

Manual Testing Scenarios

TC-001-001: User Registration and Login

Precondition:

User is on the login page with no existing account

Test Steps:
  1. Click "Create New Account" button
  2. Enter email address and password
  3. Complete registration form
  4. Log in with created credentials
Expected Result:

User account is created successfully and user is logged in to the dashboard

FR-002: Create and Manage Backlog Items (Tasks)

Description: The system shall allow users to create, view, update, and delete backlog items (tasks). Each task should support title, description, and additional metadata.

Related User Role: All Users

Manual Testing Scenarios

TC-002-001: Create New Task

Precondition:

User is logged in and viewing the backlog dashboard

Test Steps:
  1. Click "Add New Task" button
  2. Enter task title and description
  3. Click "Save" button
Expected Result:

New task is created and appears in the backlog list

FR-003: Prioritization, Sorting, and Status Management

Description: The system shall enable users to prioritize backlog items, sort them by various attributes, and manage their status (e.g., Open, In Progress, Completed). Users should be able to reorder items by priority and sort by other criteria such as priority level, creation date, or status.

Related User Role: All Users

Manual Testing Scenarios

TC-003-001: Sort Items by Priority

Precondition:

User has multiple backlog items with different priority levels in the system

Test Steps:
  1. Navigate to backlog list view
  2. Click on "Sort by Priority" option
  3. Select "High to Low" sort order
  4. Verify items are displayed in priority order
Expected Result:

Backlog items are sorted by priority with highest priority items displayed first

FR-004: Search and Filter Functionality

Description: The system shall provide comprehensive search and filtering capabilities to help users find and organize specific backlog items based on title, status, priority, or other attributes. Users should be able to apply multiple filters simultaneously and refine their search results.

Related User Role: All Users

Manual Testing Scenarios

TC-004-001: Filter Backlog Items by Status

Precondition:

User has backlog items with different statuses (Open, In Progress, Completed)

Test Steps:
  1. Navigate to backlog list view
  2. Click on status filter dropdown
  3. Select "In Progress" status
  4. Verify only "In Progress" items are displayed
Expected Result:

Only backlog items with "In Progress" status are displayed in the list

TC-004-002: Search Backlog Items

Precondition:

User has multiple backlog items with varied titles and descriptions

Test Steps:
  1. Navigate to backlog list view
  2. Click on search box
  3. Enter search keyword (e.g., "design" or "API")
  4. Verify matching items are displayed
Expected Result:

Search returns backlog items matching the search term in title or description
