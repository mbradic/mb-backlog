mb-backlog - Functional Requirements v1.0

The following section details the specific functional requirements for mb-backlog v1.0. Each requirement is assigned a unique identifier for traceability. This is a single-user personal task management application designed to provide a centralized platform for managing personal work items.

List of Functional Requirements

ID Feature Title Description
FR-001 Create Personal Tasks User can create new personal tasks with title and description
FR-002 Manage Task Status User can update task status (New, In Progress, Completed)
FR-003 Filter and Search Tasks User can filter and search tasks by status and keywords
FR-004 Sort Work Items User can sort tasks by date created, status, and priority
FR-005 View Personal Backlog User can view all tasks in a centralized backlog view

FR-001: Create Personal Tasks

Description: The system shall allow the user to create new personal tasks with a title and description. Each task represents a work item in the user's personal backlog.

Related User Role: Individual Task Manager

Manual Testing Scenarios

TC-001-001: Create Task with Title and Description

Precondition:

User is logged into the mb-backlog application and is on the main backlog dashboard.

Test Steps:
Step 1: User clicks the "Create Task" or "Add New Task" button
Step 2: System displays a task creation form with fields for Title and Description
Step 3: User enters "Implement user authentication" in the Title field
Step 4: User enters "Add login and registration functionality to the application" in the Description field
Step 5: User clicks the "Save Task" button
Expected Result:

Task is successfully created with the provided title and description. The new task appears in the backlog list with a default status of "New". User receives a confirmation message.

TC-001-002: Create Task with Title Only

Precondition:

User is on the task creation form.

Test Steps:
Step 1: User enters "Design database schema" in the Title field
Step 2: User leaves the Description field empty
Step 3: User clicks the "Save Task" button
Expected Result:

Task is created with only the title. Description field remains empty. Task is added to the backlog.

FR-002: Manage Task Status

Description: The system shall allow the user to update the status of personal tasks. Tasks can be in one of three states: New, In Progress, or Completed.

Related User Role: Individual Task Manager

Manual Testing Scenarios

TC-002-001: Change Task Status from New to In Progress

Precondition:

A task with status "New" exists in the user's backlog.

Test Steps:
Step 1: User selects a task from the backlog list
Step 2: User clicks on the status field or a status dropdown menu
Step 3: User selects "In Progress" from the status options
Step 4: System updates the task status
Expected Result:

Task status is successfully changed to "In Progress". The change is immediately reflected in the backlog view.

TC-002-002: Mark Task as Completed

Precondition:

A task with status "In Progress" exists in the user's backlog.

Test Steps:
Step 1: User selects a task that is in progress
Step 2: User clicks on the status dropdown and selects "Completed"
Step 3: System updates the task status to Completed
Expected Result:

Task status is changed to "Completed". The task may be visually distinguished (e.g., strikethrough, grayed out) in the backlog list.

FR-003: Filter and Search Tasks

Description: The system shall provide filtering and search capabilities that allow the user to find specific tasks by status and keywords. This enables the user to focus on relevant work items.

Related User Role: Individual Task Manager

Manual Testing Scenarios

TC-003-001: Filter Tasks by Status

Precondition:

User has multiple tasks with different statuses in the backlog.

Test Steps:
Step 1: User locates the filter panel or filter options in the backlog view
Step 2: User selects "In Progress" status filter checkbox
Step 3: System filters the backlog to show only tasks with "In Progress" status
Expected Result:

Only tasks with "In Progress" status are displayed in the backlog. Other tasks are hidden from view.

TC-003-002: Search Tasks by Keyword

Precondition:

User has multiple tasks in the backlog with different titles and descriptions.

Test Steps:
Step 1: User locates the search input field in the backlog view
Step 2: User types "database" in the search field
Step 3: System searches through task titles and descriptions for matches
Expected Result:

System displays only tasks that contain "database" in their title or description. All matching results are highlighted or clearly visible.

FR-004: Sort Work Items

Description: The system shall provide sorting capabilities that allow the user to organize tasks by date created, status, and priority. This helps the user view tasks in their preferred order.

Related User Role: Individual Task Manager

Manual Testing Scenarios

TC-004-001: Sort Tasks by Date Created (Newest First)

Precondition:

User has multiple tasks in the backlog with different creation dates.

Test Steps:
Step 1: User locates the sort options in the backlog view
Step 2: User selects "Date Created (Newest First)" from the sort dropdown
Step 3: System reorganizes tasks in the backlog by creation date in descending order
Expected Result:

Tasks are sorted with the most recently created tasks appearing first in the list.

TC-004-002: Sort Tasks by Status

Precondition:

User has tasks with different statuses (New, In Progress, Completed) in the backlog.

Test Steps:
Step 1: User selects "Sort by Status" from the sort options
Step 2: System reorganizes tasks grouped by their status
Expected Result:

Tasks are sorted by status, typically in order: New, In Progress, Completed. Tasks with the same status maintain their relative order.

FR-005: View Personal Backlog

Description: The system shall provide a centralized backlog view where the user can see all their personal tasks. This view serves as the primary interface for personal task management and organization.

Related User Role: Individual Task Manager

Manual Testing Scenarios

TC-005-001: Display All Tasks in Backlog View

Precondition:

User is logged into mb-backlog and has at least 5 tasks in the backlog.

Test Steps:
Step 1: User navigates to the main backlog dashboard
Step 2: System loads and displays the backlog view
Step 3: User verifies that all tasks are visible in a list or table format
Expected Result:

All personal tasks are displayed in the backlog view with their title, description (if available), status, and creation date clearly visible. The view is organized and easy to read.

TC-005-002: View Empty Backlog

Precondition:

User has no tasks in the backlog.

Test Steps:
Step 1: User navigates to the backlog view
Step 2: System loads the backlog view with no tasks
Expected Result:

System displays a message such as "No tasks found. Create a new task to get started" or an empty state UI with a call-to-action button to create the first task.
